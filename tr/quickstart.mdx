description: "Ribaunt'u kurun, gizli anahtarınızı ayarlayın, challenge ve doğrulama için iki sunucu uç noktası ekleyin ve tarayıcı widget'ını birkaç dakika içinde sayfanıza ekleyin."
---

Bu rehber, eksiksiz bir Ribaunt CAPTCHA entegrasyonu oluşturma sürecinde size eşlik edecek: proof-of-work challenge'larını yayınlayan ve doğrulayan iki sunucu uç noktası ile kullanıcılarınızın etkileşimde bulunduğu bir tarayıcı widget'ı. Sonunda, herhangi bir formun veya API işleminin önüne koyabileceğiniz çalışan bir bot koruma katmanına sahip olacaksınız.

<Steps>
  <Step title="Ribaunt'u kurun">
    `ribaunt` paketini projenize ekleyin.

    <CodeGroup>

    ```bash npm
    npm install ribaunt
    ```

    ```bash yarn
    yarn add ribaunt
    ```

    ```bash pnpm
    pnpm add ribaunt
    ```

    </CodeGroup>
  </Step>
  <Step title="Gizli anahtarınızı ayarlayın">
    Ribaunt, her challenge token'ını sizin kontrol ettiğiniz bir gizli anahtarla imzalar. Sunucu ortamınıza `RIBAUNT_SECRET` ekleyin — bir `.env` dosyası, barındırma platformunuzun gizli anahtar yöneticisi ya da sunucu yapılandırmanızı nasıl yönetiyorsanız.

    ```env
    RIBAUNT_SECRET="replace-with-a-long-random-secret"
    ```

    <Warning>
      `RIBAUNT_SECRET` değerini asla tarayıcıya ifşa etmeyin. Next.js'te bu değişkenin önüne **kesinlikle** `NEXT_PUBLIC_` getirmeyin — bu durum gizli anahtarı istemci paketine gömerek herkesin geçerli challenge token'ları üretmesine olanak tanır.
    </Warning>
  </Step>
  <Step title="Sunucu uç noktalarını oluşturun">
    İki uç noktaya ihtiyacınız var: biri challenge yayınlayan, diğeri çözümleri doğrulayan. Aşağıda `ribaunt` paketindeki `createChallenge` ve `verifySolution` fonksiyonlarını kullanan eksiksiz bir Express örneği yer alıyor.

    ```ts
    import 'dotenv/config';
    import express from 'express';
    import { createChallenge, verifySolution } from 'ribaunt';
    
    const app = express();
    app.use(express.json());
    
    // Issues signed JWT challenges to the browser widget
    app.get('/api/captcha/challenge', (_req, res) => {
      const challenges = createChallenge(5, 4, 120);
      // createChallenge(difficulty, amount, ttlSeconds)
      res.json({ challenges });
    });
    
    // Verifies the nonces submitted by the browser widget
    app.post('/api/captcha/verify', async (req, res) => {
      const { tokens, solutions } = req.body;
      const valid = await verifySolution(tokens, solutions);
    
      if (!valid) {
        return res.status(400).json({ success: false, error: 'Invalid CAPTCHA solution' });
      }
    
      return res.json({ success: true });
    });
    
    app.listen(3000);
    ```

    `createChallenge` üç parametre alır:

    | Parametre | Varsayılan | Açıklama |
    | --- | --- | --- |
    | `difficulty` | `5` | SHA-256 hash'inde gereken öndeki sıfır sayısı. Daha yüksek değerler çözüm süresini artırır. |
    | `amount` | `4` | Oluşturulacak challenge token sayısı. |
    | `ttlSeconds` | `30` | Challenge token'ının saniye cinsinden ömrü. |

    `verifySolution` bir `Promise<boolean>` döndürür. JWT imzasını doğrular, süre dolumunu kontrol eder, nonce'un gerekli öndeki sıfırlara sahip bir hash ürettiğini onaylar ve varsayılan olarak süreç-yerel depoyu kullanarak replay'i engeller.
  </Step>
  <Step title="Widget'ı ön yüzünüze ekleyin">
    Ribaunt web bileşeni script'ini ekleyin ve CAPTCHA koruması istediğiniz yere `<ribaunt-widget>` öğesini yerleştirin. Widget sunucunuzdan bir challenge alır, arka planda çözer ve `auto-verify="true"` ayarlandığında doğrulama uç noktanızı otomatik olarak çağırır.

    ```html
    <script type="module" src="/node_modules/ribaunt/dist/widget-browser.js"></script>
    
    <ribaunt-widget
      challenge-endpoint="/api/captcha/challenge"
      verify-endpoint="/api/captcha/verify"
      auto-verify="true"
    ></ribaunt-widget>
    
    <script>
      const widget = document.querySelector('ribaunt-widget');
    
      widget.addEventListener('verify', (e) => {
        console.log('Verified!', e.detail.solutions);
      });
    </script>
    ```

    React kullanıyorsanız web bileşenini doğrudan kullanmak yerine sarmalayıcı bileşeni içe aktarın. Tam örnek için [React entegrasyonu](/tr/integrations/react) sayfasına bakın.
  </Step>
</Steps>

<Note>
  Widget, challenge çözüldüğünde bir `verify` olayı yayar. Bunu gönder butonunuzu etkinleştirmek veya form göndermeye devam etmek için kullanın. Doğrulama başarısız olursa, kullanıcıya geri bildirim sunma fırsatı veren bir `error` olayı tetiklenir.
</Note>
