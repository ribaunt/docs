Ribaunt'un `ribaunt/widget-react` adresindeki React sarmalayıcısı, yalnızca tarayıcıda çalışan web bileşenini istemcide dinamik olarak içe aktararak SSR korumalarını otomatik olarak halleder. Bileşen dosyanıza `'use client'` ekleyin, `RibauntWidget` bileşenini içe aktarın ve hazırsınız — ek bir yapılandırma gerekmez.

<Warning>
  **`RIBAUNT_SECRET` değerinin önüne asla `NEXT_PUBLIC_` getirmeyin.**\
  Bunu yapmak gizli anahtarı istemci paketinize gömerek her ziyaretçiye ifşa eder. `RIBAUNT_SECRET` değerini `NEXT_PUBLIC_` öneki olmadan `.env.local` dosyasında tutun ki yalnızca sunucuda kalsın.
</Warning>

## Client component kurulumu

Bir client component oluşturun ve `RibauntWidget` bileşenini yerleştirin. Sarmalayıcı dinamik içe aktarmayı dahili olarak hallettiği için, **`next/dynamic` ile sarmalamanıza gerek yoktur**.

```tsx
'use client';

import { useRef } from 'react';
import RibauntWidget, { type RibauntWidgetHandle } from 'ribaunt/widget-react';

export default function ContactForm() {
  const widgetRef = useRef<RibauntWidgetHandle>(null);

  return (
    <form>
      <RibauntWidget
        ref={widgetRef}
        challengeEndpoint="/api/captcha/challenge"
        verifyEndpoint="/api/captcha/verify"
        autoVerify={true}
        solveTimeout={15000}
        onVerify={(detail) => console.log('Verified:', detail.solutions)}
        onError={(detail) => console.error('Failed:', detail.error)}
        onReady={(detail) => console.log('Ready:', detail.state)}
      />
      <button type="submit">Submit</button>
    </form>
  );
}
```

<Note>
  **`next/dynamic` ile sarmalamayın** — `ribaunt/widget-react` sarmalayıcısı dinamik içe aktarmayı dahili olarak halleder. Ekstra bir `next/dynamic` çağrısı gereksizdir ve çift yüklemeye neden olabilir.
</Note>

## API rotaları

Yukarıdaki client component, `/api/captcha/challenge` ve `/api/captcha/verify` uç noktalarını çağırır. Challenge üretmek ve çözümleri doğrulamak için bu sunucu tarafı uç noktalarını oluşturmanız gerekir. Tam rota işleyici uygulaması için [Next.js sunucu rehberine](/tr/server/nextjs) bakın.

## Önemli noktalar

- **Daima `'use client'` kullanın** — `RibauntWidget` tarayıcıda render olur ve tarayıcı API'lerine dayanır.
- **`next/dynamic` kullanmayın** — sarmalayıcı dinamik içe aktarmayı tek başına halleder; `next/dynamic` eklemek gereksizdir.
- **`RIBAUNT_SECRET` değerini `NEXT_PUBLIC_` ile ifşa etmeyin** — gizli anahtar yalnızca sunucu tarafında kalmalıdır.
- **HTTPS veya localhost kullanın** — tarayıcıda çözüm, güvenli bağlamlarla sınırlandırılmış olan Web Crypto API'sine bağlıdır. Düz yerel ağ HTTP URL'leri (örneğin `http://192.168.x.x`) çalışmaz.
- **Sunucusuz veya edge dağıtımları için** — varsayılan replay önleme süreç-yerel belleği kullanır ve bu, sunucusuz örnekler arasında paylaşılmaz. Bu ortamlarda replay saldırılarını önlemek için atomik bir dağıtık depo ile `replayPrevention: 'remote'` ayarını kullanın.
