Ribaunt'un `ribaunt/widget-react`'teki React sarmalayıcısı, yalnızca tarayıcıda çalışan web bileşenini istemcide dinamik olarak içe aktararak SSR korumalarını otomatik olarak yönetir. Bileşen dosyanıza `'use client'` ekleyin, `RibauntWidget`'i içe aktarın ve hazırsınız — ek yapılandırma gerekmez.

<Warning>
  **`RIBAUNT_SECRET`'in başına asla `NEXT_PUBLIC_` eklemeyin.**  
  Bunu yapmak gizli anahtarı istemci paketinize gömer ve her ziyaretçiye açar. `RIBAUNT_SECRET`'i yalnızca sunucuda kalması için `NEXT_PUBLIC_` ön eki olmadan `.env.local`'da tutun.
</Warning>

## İstemci bileşeni kurulumu

Bir istemci bileşeni oluşturun ve `RibauntWidget`'i ekleyin. Sarmalayıcı dinamik içe aktarmayı dahili olarak yönetdiği için `next/dynamic` içinde sarmanıza **gerek yoktur**.

```tsx
'use client';

import { useRef } from 'react';
import RibauntWidget, { type RibauntWidgetHandle } from 'ribaunt/widget-react';

export default function ContactForm() {
  const widgetRef = useRef<RibauntWidgetHandle>(null);

  return (
    <form>
      <RibauntWidget
        ref={widgetRef}
        challengeEndpoint="/api/captcha/challenge"
        verifyEndpoint="/api/captcha/verify"
        autoVerify={true}
        solveTimeout={15000}
        onVerify={(detail) => console.log('Verified:', detail.solutions)}
        onError={(detail) => console.error('Failed:', detail.error)}
        onReady={(detail) => console.log('Ready:', detail.state)}
      />
      <button type="submit">Submit</button>
    </form>
  );
}
```

<Note>
  **`next/dynamic` içinde SARMAYIN** — `ribaunt/widget-react` sarmalayıcısı dinamik içe aktarmayı zaten dahili olarak yönetir. Ek bir `next/dynamic` çağrısı gereksizdir ve çift yüklemeye neden olabilir.
</Note>

## API rotaları

Yukarıdaki istemci bileşeni `/api/captcha/challenge` ve `/api/captcha/verify` çağırır. Meydan okumalar oluşturmak ve çözümleri doğrulamak için bu sunucu tarafı uç noktalarını oluşturmanız gerekir. Tam rota işleyici uygulaması için [Next.js sunucu kılavuzuna](/tr/server/nextjs) bakın.

## Önemli noktalar

- **Her zaman `'use client'` kullanın** — `RibauntWidget` tarayıcıda render edilir ve tarayıcı API'lerine dayanır.
- **`next/dynamic` kullanmayın** — sarmalayıcı dinamik içe aktarmayı kendisi yönetir; `next/dynamic` eklemek gereksizdir.
- **`RIBAUNT_SECRET`'i `NEXT_PUBLIC_` ile açmayın** — gizli anahtar yalnızca sunucu tarafında kalmalıdır.
- **HTTPS veya localhost kullanın** — tarayıcı çözümü, güvenli bağlamlarla sınırlı olan Web Crypto API'sine bağlıdır. Düz yerel ağ HTTP URL'leri (örn. `http://192.168.x.x`) çalışmaz.
- **Sunucusuz veya edge dağıtımları için** — varsayılan yeniden oynatma önleme, sunucusuz örnekler arasında paylaşılmayan işlem yerel belleği kullanır. Bu ortamlarda yeniden oynatma saldırılarını önlemek için atomik bir dağıtık depoyla `replayPrevention: 'remote'` ayarlayın.
