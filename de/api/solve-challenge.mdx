`solveChallenge()` führt denselben Proof-of-Work-Algorithmus aus, den das Browser-Widget verwendet, jedoch synchron in Node.js. Es ist für das automatisierte Testen Ihrer Challenge- und Verify-Endpoints konzipiert — nicht für den produktiven Einsatz in Request-Handlern.

## Import

```ts
import { solveChallenge } from 'ribaunt';
```

## Signatur

```ts
// Single token
function solveChallenge(
  token: ChallengeToken,
  options?: SolveChallengeOptions
): ChallengeSolution | undefined

// Array of tokens
function solveChallenge(
  token: ChallengeToken[],
  options?: SolveChallengeOptions
): ChallengeSolution[] | undefined
```

## Parameter

<ParamField path="token" type="ChallengeToken | ChallengeToken[]" required>
  Ein einzelnes JWT-Challenge-Token oder ein Array von Token aus `createChallenge()`.
</ParamField>

<ParamField path="options" type="SolveChallengeOptions">
  Optionale Schutzmechanismen, um lang laufende synchrone Solves zu verhindern.

  <Expandable title="SolveChallengeOptions">
    <ParamField path="maxIterations" type="number">
      Harte Obergrenze für Nonce-Versuche pro Token. Gibt `undefined` zurück, wenn erreicht.
    </ParamField>

    <ParamField path="maxDurationMs" default="30000" type="number">
      Maximale synchrone Lösungszeit in Millisekunden pro Token. Gibt `undefined` zurück, wenn überschritten.
    </ParamField>
  </Expandable>
</ParamField>

## Rückgabewert

Gibt eine `ChallengeSolution` (`{ nonce: string; hash: string }`) für eine einzelne Token-Eingabe oder `ChallengeSolution[]` für eine Array-Eingabe zurück. Gibt `undefined` zurück, wenn ein Guardrail erreicht wird oder ein Token ungültig ist. Beim Lösen eines Arrays wird `undefined` zurückgegeben, sobald ein einzelnes Token fehlschlägt — es werden keine partiellen Ergebnisse zurückgegeben.

## Beispiel

```ts
import { createChallenge, solveChallenge, verifySolution } from 'ribaunt';

// Use low difficulty in tests for speed
const tokens = createChallenge(3, 2, 60);
const solutions = solveChallenge(tokens);

if (solutions) {
  const valid = await verifySolution(tokens, solutions);
  console.log('Valid:', valid); // true
}
```

Mit Schutzmechanismen:

```ts
const solution = solveChallenge(token, {
  maxDurationMs: 2000,
  maxIterations: 500_000,
});

if (!solution) {
  console.log('Solver gave up — difficulty too high or timeout reached');
}
```

<Warning>
  `solveChallenge` ist synchron und CPU-intensiv. Rufen Sie es niemals in einem Produktions-HTTP-Request-Handler auf — es blockiert Ihre Node.js-Event-Loop.
</Warning>

<Tip>
  Verwenden Sie in Tests Schwierigkeit 3–4. Schwierigkeit 5 verlangsamt Ihre Testsuite spürbar.
</Tip>
