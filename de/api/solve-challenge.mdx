title: "solveChallenge — Synchroner PoW-Solver für Tests"
sidebarTitle: "solveChallenge"
description: "solveChallenge(token, options?) löst JWT-Challenge-Tokens synchron. Verwenden Sie es in Tests und Tools — nicht in produktiven Request-Handlern."
---

`solveChallenge()` führt denselben Proof-of-Work-Algorithmus aus wie das Browser-Widget, jedoch synchron in Node.js. Es ist für automatisierte Tests Ihrer Challenge- und Verify-Endpunkte gedacht — nicht für den produktiven Einsatz in Request-Handlern.

## Import

```ts
import { solveChallenge } from 'ribaunt';
```

## Signatur

```ts
// Single token
function solveChallenge(
  token: ChallengeToken,
  options?: SolveChallengeOptions
): ChallengeSolution | undefined

// Array of tokens
function solveChallenge(
  token: ChallengeToken[],
  options?: SolveChallengeOptions
): ChallengeSolution[] | undefined
```

## Parameter

<ParamField path="token" type="ChallengeToken | ChallengeToken[]" required>
  Ein einzelnes JWT-Challenge-Token oder ein Array von Tokens aus `createChallenge()`.
</ParamField>

<ParamField path="options" type="SolveChallengeOptions">
  Optionale Schutzgrenzen, um lang laufende synchrone Lösungen zu verhindern.

  <Expandable title="SolveChallengeOptions">
    <ParamField path="maxIterations" type="number">
      Harte Obergrenze für Nonce-Versuche pro Token. Gibt `undefined` zurück, wenn erreicht.
    </ParamField>

    <ParamField path="maxDurationMs" type="number" default="30000">
      Maximale synchrone Lösungszeit in Millisekunden pro Token. Gibt `undefined` zurück, wenn überschritten.
    </ParamField>
  </Expandable>
</ParamField>

## Rückgabewert

Gibt eine `ChallengeSolution` (`{ nonce: string; hash: string }`) für eine Einzeltoken-Eingabe zurück oder `ChallengeSolution[]` für ein Array. Gibt `undefined` zurück, wenn eine Schutzgrenze erreicht wird oder ein Token ungültig ist. Beim Lösen eines Arrays wird `undefined` zurückgegeben, sobald ein einzelnes Token fehlschlägt — es werden keine Teilergebnisse zurückgegeben.

## Beispiel

```ts
import { createChallenge, solveChallenge, verifySolution } from 'ribaunt';

// Use low difficulty in tests for speed
const tokens = createChallenge(3, 2, 60);
const solutions = solveChallenge(tokens);

if (solutions) {
  const valid = await verifySolution(tokens, solutions);
  console.log('Valid:', valid); // true
}
```

Mit Schutzgrenzen:

```ts
const solution = solveChallenge(token, {
  maxDurationMs: 2000,
  maxIterations: 500_000,
});

if (!solution) {
  console.log('Solver gave up — difficulty too high or timeout reached');
}
```

<Warning>
  `solveChallenge` ist synchron und CPU-intensiv. Rufen Sie es niemals in einem produktiven HTTP-Request-Handler auf — es würde Ihren Node.js-Event-Loop blockieren.
</Warning>

<Tip>
  Verwenden Sie in Tests Schwierigkeit 3–4. Schwierigkeit 5 verlangsamt Ihre Test-Suite spürbar.
</Tip>
