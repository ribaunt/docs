`createChallenge()` wird aus `ribaunt` importiert und serverseitig aufgerufen, um einen oder mehrere Proof-of-Work-Challenge-Token zu generieren. Jedes Token ist ein signiertes JWT, das der Browser-Solver dekodiert und löst.

## Import

```ts
import { createChallenge } from 'ribaunt';
```

## Signatur

```ts
function createChallenge(
  difficulty?: number,
  amount?: number,
  ttlSeconds?: number
): ChallengeToken[]
```

## Parameter

<ParamField path="difficulty" default="5" type="number">
  Anzahl der führenden Null-Hex-Ziffern, die im SHA-256-Hash erforderlich sind. Jede Erhöhung verdoppelt etwa die Lösungszeit. Werte über 6 können dazu führen, dass Browser hängen bleiben.
</ParamField>

<ParamField path="amount" default="4" type="number">
  Anzahl der zu generierenden Challenge-Token. Mehr Challenges erhöhen den gesamten Proof-of-Work, aber auch die Netzwerkbandbreite.
</ParamField>

<ParamField path="ttlSeconds" default="30" type="number">
  Lebensdauer des Challenge-Tokens in Sekunden. Nach Ablauf eingereichte Token werden von `verifySolution` abgelehnt.
</ParamField>

## Rückgabewert

Gibt `ChallengeToken[]` zurück — ein Array signierter JWT-Strings. Senden Sie dieses Array als `{ challenges: tokens }` an den Browser.

## Beispiel

```ts
import { createChallenge } from 'ribaunt';

// Fast: low difficulty, short TTL (login forms, comment boxes)
const fast = createChallenge(4, 4, 30);

// Moderate: standard configuration for most use cases
const moderate = createChallenge(5, 4, 120);

// High: sensitive actions (account creation, password reset)
const high = createChallenge(5, 8, 120);
```

## Empfohlene Einstellungen

| Anwendungsfall | Difficulty | Amount | TTL |
| --- | --- | --- | --- |
| Schnell / Hintergrund | 4 | 4 | 30 |
| Moderat / Formularübermittlung | 5 | 4 | 60 |
| Hoch / sensible Aktionen | 5 | 8 | 120 |

## Validierung

`createChallenge()` validiert alle drei Parameter zur Laufzeit und wirft einen Fehler, wenn ein Wert ungültig ist:

- **`difficulty`** — muss eine endliche Zahl und mindestens `1` sein. Bruchwerte werden mit `Math.floor()` abgerundet.
- **`amount`** — muss eine endliche Zahl und mindestens `1` sein. Bruchwerte werden mit `Math.floor()` abgerundet.
- **`ttlSeconds`** — muss eine endliche Zahl und mindestens `1` sein. Bruchwerte werden mit `Math.floor()` abgerundet.

<Warning>
  Lassen Sie niemals benutzergesteuerte Request-Parameter ohne Validierung direkt in `createChallenge()` einfließen.
</Warning>

<Note>
  Erfordert, dass `RIBAUNT_SECRET` als Umgebungsvariable gesetzt ist. `createChallenge()` wirft einen Fehler, wenn das Secret fehlt.
</Note>
