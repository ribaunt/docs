Diese Anleitung führt dich durch den Aufbau einer vollständigen Ribaunt-CAPTCHA-Integration: zwei Server-Endpunkte, die Proof-of-Work-Challenges ausstellen und verifizieren, sowie ein Browser-Widget, mit dem deine Nutzer interagieren. Am Ende verfügst du über eine funktionierende Bot-Schutzschicht, die du vor jedes Formular oder jede API-Aktion setzen kannst.

<Steps>
  <Step title="Ribaunt installieren">
    Füge das `ribaunt`-Paket zu deinem Projekt hinzu.

    <CodeGroup>

    ```bash npm
    npm install ribaunt
    ```

    ```bash yarn
    yarn add ribaunt
    ```

    ```bash pnpm
    pnpm add ribaunt
    ```

    </CodeGroup>
  </Step>
  <Step title="Secret festlegen">
    Ribaunt signiert jedes Challenge-Token mit einem Secret, das du kontrollierst. Füge `RIBAUNT_SECRET` zu deiner Serverumgebung hinzu – über eine `.env`-Datei, den Secrets-Manager deiner Hosting-Plattform oder wie auch immer du deine Serverkonfiguration verwaltest.

    ```env
    RIBAUNT_SECRET="replace-with-a-long-random-secret"
    ```

    <Warning>
      Lege `RIBAUNT_SECRET` niemals im Browser offen. In Next.js solltest du es **nicht** mit `NEXT_PUBLIC_` präfixieren – das würde das Secret in dein Client-Bundle einbetten und es jedem ermöglichen, gültige Challenge-Tokens zu fälschen.
    </Warning>
  </Step>
  <Step title="Server-Endpunkte erstellen">
    Du benötigst zwei Endpunkte: einen, der Challenges ausstellt, und einen, der Lösungen verifiziert. Hier ist ein vollständiges Express-Beispiel mit den aktuellen APIs von `createChallenge` und `verifySolution` aus `ribaunt`.

    ```ts
    import 'dotenv/config';
    import express from 'express';
    import { createChallenge, verifySolution } from 'ribaunt';

    const app = express();
    app.use(express.json());

    app.get('/api/captcha/challenge', (_req, res) => {
      const challenges = createChallenge({
        difficulty: 5,
        amount: 4,
        ttlSeconds: 120,
        context: 'signup',
      });

      res.json({ challenges });
    });

    app.post('/api/captcha/verify', async (req, res) => {
      const { tokens, solutions } = req.body;
      const result = await verifySolution(tokens, solutions, { context: 'signup' });

      if (!result.valid) {
        return res.status(400).json({ success: false, error: result.reason, message: result.message });
      }

      return res.json({ success: true });
    });

    app.listen(3000);
    ```

    `createChallenge()` akzeptiert sowohl Positionsargumente als auch ein Optionsobjekt. Das Optionsobjekt unterstützt `difficulty`, `amount`, `ttlSeconds`, `context` und eine adaptive `workload`-Konfiguration.

    `verifySolution()` gibt nun ein strukturiertes Ergebnisobjekt zurück statt eines bloßen Booleans. Prüfe `result.valid` und verwende `result.reason` oder `result.message`, wenn die Verifizierung fehlschlägt.
  </Step>
  <Step title="Widget in dein Frontend einbinden">
    Binde das Ribaunt-Web-Component-Skript ein und platziere das `<ribaunt-widget>`-Element überall dort, wo du CAPTCHA-Schutz benötigst. Das Widget lädt eine Challenge von deinem Server, löst sie im Hintergrund und sendet die Lösungen anschließend an deinen Verify-Endpunkt, wenn `auto-verify="true"` gesetzt ist.

    ```html
    <script type="module" src="/node_modules/ribaunt/dist/widget-browser.js"></script>

    <ribaunt-widget
      challenge-endpoint="/api/captcha/challenge"
      verify-endpoint="/api/captcha/verify"
      auto-verify="true"
      solve-timeout="15000"
    ></ribaunt-widget>

    <script>
      const widget = document.querySelector('ribaunt-widget');

      widget.addEventListener('verify', (event) => {
        console.log('Verified!', event.detail.solutions);
      });
    </script>
    ```

    Wenn du React verwendest, importiere stattdessen die Wrapper-Komponente anstelle der Web-Komponente direkt. Das vollständige Beispiel findest du unter [React-Integration](/integrations/react).
  </Step>
</Steps>

<Note>
  Das Widget sendet ein `verify`-Event, sobald die Challenge gelöst ist. Ein entsprechendes `error`-Event wird ausgelöst, wenn die Verifizierung fehlschlägt, und `state-change` meldet Zwischenzustände wie `fetching`, `solving`, `verifying` und `done`.
</Note>
