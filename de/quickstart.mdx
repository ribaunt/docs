Diese Anleitung führt Sie durch den Aufbau einer vollständigen Ribaunt-CAPTCHA-Integration: zwei Server-Endpunkte, die Proof-of-Work-Challenges ausstellen und verifizieren, sowie ein Browser-Widget, mit dem Ihre Benutzer interagieren. Am Ende verfügen Sie über eine funktionierende Bot-Schutzschicht, die Sie vor jedes Formular oder jede API-Aktion setzen können.

<Steps>
  <Step title="Ribaunt installieren">
    Fügen Sie das Paket `ribaunt` zu Ihrem Projekt hinzu.

    <CodeGroup>

    ```bash npm
    npm install ribaunt
    ```

    ```bash yarn
    yarn add ribaunt
    ```

    ```bash pnpm
    pnpm add ribaunt
    ```

    </CodeGroup>
  </Step>
  <Step title="Secret setzen">
    Ribaunt signiert jedes Challenge-Token mit einem Secret, das Sie kontrollieren. Fügen Sie `RIBAUNT_SECRET` zu Ihrer Server-Umgebung hinzu – einer `.env`-Datei, dem Secrets-Manager Ihres Hosting-Anbieters oder wie auch immer Sie Ihre Serverkonfiguration verwalten.

    ```env
    RIBAUNT_SECRET="replace-with-a-long-random-secret"
    ```

    <Warning>
      Geben Sie `RIBAUNT_SECRET` niemals an den Browser weiter. In Next.js dürfen Sie es **nicht** mit `NEXT_PUBLIC_` präfixieren – dadurch würde das Secret in Ihr Client-Bundle eingebettet und es jedem ermöglichen, gültige Challenge-Tokens zu fälschen.
    </Warning>
  </Step>
  <Step title="Server-Endpunkte erstellen">
    Sie benötigen zwei Endpunkte: einen, der Challenges ausstellt, und einen, der Lösungen verifiziert. Hier ein vollständiges Express-Beispiel mit den Funktionen `createChallenge` und `verifySolution` aus `ribaunt`.

    ```ts
    import 'dotenv/config';
    import express from 'express';
    import { createChallenge, verifySolution } from 'ribaunt';
    
    const app = express();
    app.use(express.json());
    
    // Issues signed JWT challenges to the browser widget
    app.get('/api/captcha/challenge', (_req, res) => {
      const challenges = createChallenge(5, 4, 120);
      // createChallenge(difficulty, amount, ttlSeconds)
      res.json({ challenges });
    });
    
    // Verifies the nonces submitted by the browser widget
    app.post('/api/captcha/verify', async (req, res) => {
      const { tokens, solutions } = req.body;
      const valid = await verifySolution(tokens, solutions);
    
      if (!valid) {
        return res.status(400).json({ success: false, error: 'Invalid CAPTCHA solution' });
      }
    
      return res.json({ success: true });
    });
    
    app.listen(3000);
    ```

    `createChallenge` akzeptiert drei Parameter:

    | Parameter | Standard | Beschreibung |
    | --- | --- | --- |
    | `difficulty` | `5` | Erforderliche führende Nullen im SHA-256-Hash. Höhere Werte verlängern die Lösungszeit. |
    | `amount` | `4` | Anzahl der zu erstellenden Challenge-Tokens. |
    | `ttlSeconds` | `30` | Lebensdauer des Challenge-Tokens in Sekunden. |

    `verifySolution` gibt ein `Promise<boolean>` zurück. Es validiert die JWT-Signatur, prüft das Ablaufdatum, bestätigt, dass die Nonce einen Hash mit den erforderlichen führenden Nullen erzeugt, und blockiert Replay-Angriffe standardmäßig über prozesslokalen Speicher.
  </Step>
  <Step title="Widget zum Frontend hinzufügen">
    Binden Sie das Ribaunt-Web-Component-Skript ein und platzieren Sie das `<ribaunt-widget>`-Element überall dort, wo Sie CAPTCHA-Schutz benötigen. Das Widget ruft eine Challenge von Ihrem Server ab, löst sie im Hintergrund und ruft anschließend automatisch Ihren Verify-Endpunkt auf, wenn `auto-verify="true"` gesetzt ist.

    ```html
    <script type="module" src="/node_modules/ribaunt/dist/widget-browser.js"></script>
    
    <ribaunt-widget
      challenge-endpoint="/api/captcha/challenge"
      verify-endpoint="/api/captcha/verify"
      auto-verify="true"
    ></ribaunt-widget>
    
    <script>
      const widget = document.querySelector('ribaunt-widget');
    
      widget.addEventListener('verify', (e) => {
        console.log('Verified!', e.detail.solutions);
      });
    </script>
    ```

    Wenn Sie React verwenden, importieren Sie stattdessen die Wrapper-Komponente anstelle der Web Component direkt. Das vollständige Beispiel finden Sie unter [React-Integration](/de/integrations/react).
  </Step>
</Steps>

<Note>
  Das Widget löst ein `verify`-Event aus, wenn die Challenge gelöst ist. Verwenden Sie es, um Ihren Submit-Button zu aktivieren oder mit der Formularübermittlung fortzufahren. Ein entsprechendes `error`-Event wird ausgelöst, wenn die Verifizierung fehlschlägt, sodass Sie dem Benutzer Feedback geben können.
</Note>
